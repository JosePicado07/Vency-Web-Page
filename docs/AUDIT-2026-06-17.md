## Anti-Patterns Verdict

**Pass — not AI-generated. Stronger than baseline.**

The site now operates in a single visual register. The Decants editorial language (full-bleed photography, per-fragrance color identity, Bitter 900 display type) has propagated to the Catalog (section headers) and Homepage (Sessions section). The Hybrid block's faint `∞` watermark removes the "placeholder" reading. Format toggle selected state in warm-ochre and WA CTAs permanently visible in catalog rows remove two of the three remaining generic web patterns.

One remaining concern: the Process section on the homepage still reads as SaaS "How It Works" — numbered steps on parchment with contained images. This is the last section that does not feel like an atelier. It was outside the scope of this fix cycle.

The Catalog at 30,000px mobile remains dense, but the display-scale section headers (Diseñador / Nicho / Ultra Nicho) now provide genuine visual rhythm and threshold moments. This is the maximum improvement achievable without introducing photography into catalog entries, which was explicitly excluded.

### P1 — Major (fix before release)

---

**[P1-A] WA CTA touch target still undersized — now always visible, still untappable**
- **Location:** `src/styles/catalogo.css` line 364 (`.cat-entry__wa`)
- **Category:** Accessibility / Responsive
- **Impact:** `.cat-entry__wa` has `padding: 0 0 0.8rem` — the tappable area is text width × ~13px. The E-7 fix promoted this link from a hidden detail panel to always-visible in the row, which is correct for discoverability. However, the touch target remained undersized. On mobile at 30,000px of catalog rows, users will repeatedly misfire on the primary lead-gen action.
- **WCAG:** 2.5.5 Target Size (AA)
- **Fix:** Change `.cat-entry__wa` padding to `0.5rem 0.75rem`. Add `display: inline-flex; align-items: center; min-height: 2.75rem`. Same fix for `.cat-entry__interp`.
- **Command:** `/impeccable adapt catalogo`

---

**[P1-B] Gender dots still color-only**
- **Location:** `src/scripts/catalogo.js` (renderEntry), `src/styles/catalogo.css` lines 277–278
- **Category:** Accessibility
- **Impact:** Unchanged from baseline. Mujer (rose) / Hombre (blue) / Unisex (teal) 5×5px dots with no shape or text label. Color-blind users cannot distinguish hombre from unisex. Legend is above the fold on desktop but scrolled off on a 30,000px mobile page.
- **WCAG:** 1.4.1 Use of Color (A)
- **Fix:** Add `<span class="sr-only">Mujer</span>` (or Hombre / Unisex) immediately after each dot element in `renderEntry()`.
- **Command:** `/impeccable harden catalogo`

---

**[P1-C] No responsive images — all pages**
- **Location:** All `<img>` tags across index.html, decants.html; JS-rendered in coleccion.js
- **Category:** Performance
- **Impact:** Unchanged from baseline. 900–1400px images served to 390px mobile screens. No srcset, no WebP, no AVIF. This is the single biggest performance liability. On a 4G connection, the Decants page (6 large images) and Homepage hero add multiple seconds to LCP.
- **Fix:** Generate 400w / 800w / 1200w variants. Add `srcset` and `sizes`. Convert to WebP minimum, AVIF preferred. For coleccion.js, add `srcset` to the `renderEntry()` img tag.
- **Command:** `/impeccable optimize`

---

**[P1-D] Catalog filter bar border-left orphan on mobile — still present**
- **Location:** `src/styles/catalogo.css` lines 69–73 (`.cat-filters__group + .cat-filters__group`)
- **Category:** Responsive
- **Impact:** Unchanged from baseline. The Género group uses `border-left` as a separator. When the filter bar wraps at narrow widths, this renders as a floating vertical rule disconnected from any element. Looks broken on 390px.
- **Fix:** At the filter bar wrap breakpoint (≤520px), switch the second group separator from `border-left` to `border-top`, or stack groups vertically with `flex-direction: column`.
- **Command:** `/impeccable adapt catalogo`

### P2 — Minor

---

**[P2-A] format-rail.css has unresolved inline OKLCH values**
- **Location:** `src/styles/format-rail.css` lines 41, 90, 94, 98, 111, 181
- **Category:** Theming
- **Impact:** Group B replaced inline values in catalogo.css, decants.css, and styles.css, but format-rail.css was not in scope. Six inline values remain:
  - `oklch(68% 0.09 55)` (line 111) — near-identical to `--warm-ochre-light: oklch(70% 0.09 55)`, should be tokenized
  - `oklch(22% 0.035 180)` (line 98) — close to `--volcanic-ink` but slightly lighter; needs a named token or alignment
  - `oklch(28% 0.03 180)` (lines 90, 94, 181) — border color; no named token currently
  - `oklch(92% 0.006 80 / 0.5)` (line 41) — toggle panel bg tint; contextual, no token needed
- **Fix:** Replace `oklch(68% 0.09 55)` → `var(--warm-ochre-light)`. Align `oklch(22% 0.035 180)` with `--volcanic-ink`. Add `--border-dark: oklch(28% 0.03 180)` to `:root` for the recurring border value.
- **Command:** `/impeccable harden`

**[P2-B] Dead CSS: `.cat-entry__consult` rules**
- **Location:** `src/styles/catalogo.css` lines 280–323
- **Category:** Anti-Pattern (dead code)
- **Impact:** The `.cat-entry__consult` toggle button was removed from the JS render (E-7), but its CSS rules remain. ~43 lines of rules targeting an element that no longer exists in the DOM: opacity transitions, hover states, icon styles, expanded states. No user impact, but adds confusion for future maintainers.
- **Fix:** Delete `.cat-entry__consult`, `.cat-entry__consult:hover`, `.cat-entry__consult[aria-expanded]`, `.cat-entry__consult-icon`, `.cat-section--ultra-nicho .cat-entry__consult` rules.
- **Command:** `/impeccable distill catalogo`

**[P2-C] Dead JS: `closeAll()` and `wireEntries()` functions**
- **Location:** `src/scripts/catalogo.js` lines 123–153
- **Category:** Anti-Pattern (dead code)
- **Impact:** `wireEntries()` adds a click listener that looks for `.cat-entry__consult` — an element no longer rendered. `closeAll()` is only called from `wireEntries()`. Both functions execute on init and silently do nothing. No user impact, ~30 lines of unreachable code.
- **Fix:** Remove `closeAll()` and `wireEntries()` functions, and the `wireEntries()` call in init.
- **Command:** `/impeccable distill catalogo`

---

**[P2-D] Detail panels are permanently closed with role="region" — orphaned ARIA**
- **Location:** `src/scripts/catalogo.js` — all `cat-entry__detail` elements
- **Category:** Accessibility
- **Impact:** Each catalog entry still renders a `<div class="cat-entry__detail" role="region">` containing `.cat-entry__notes` text. The toggle to open it was removed. The region is permanently collapsed (`grid-template-rows: 0fr; overflow: hidden`) with no interactive path to open it. Screen readers encounter a `role="region"` they can announce but whose content is inaccessible. The notes text (~240 characters per entry, 100+ entries) is lost to all users.
- **Fix options (choose one):**
  - A: Remove the entire detail panel from `renderEntry()` — cleanest, eliminates dead markup
  - B: Render notes as always-visible text beneath each entry row — aligns with WA CTA promotion
  - C: Re-introduce a keyboard-accessible toggle (keeping notes accessible without the "Ver +" button)
- **Command:** `/impeccable harden catalogo`

**[P2-E] Remaining contextual inline OKLCH values in styles.css**
- **Location:** `src/styles/styles.css` lines 625, 633, 838, 1090, 1131, 1135, 1144, 1156, 1160, 1168, 1213
- **Category:** Theming
- **Impact:** 11 inline OKLCH values in styles.css were left after Group B because they had no clean named token equivalent:
  - `oklch(58% 0.015 180)` — Hybrid desc text, between `--ink-deep` and `--ink-muted`
  - `oklch(80% 0.09 55)` — WA link hover, lighter than `--warm-ochre-light`
  - `oklch(24% 0.03 180)` — mobile nav bg (near `--volcanic-ink-hover`)
  - Format toggle variant borders and hover backgrounds
- These are structural UI values, not brand color violations. Minor drift risk if tokens are adjusted.
- **Fix:** Add `--warm-ochre-pale: oklch(80% 0.09 55)` and align remaining values to nearest tokens. Non-urgent.
- **Command:** `/impeccable harden`

**[P2-F] Catalog mobile page height increased from 24,270px to 30,004px**
- **Location:** Catalog page overall
- **Category:** Responsive / UX
- **Impact:** The E-7 fix (WA links always visible in rows) and C fix (display-scale section headers) both added vertical height. The catalog is now 24% longer on mobile. Each fragrance row now renders a CTA link below the name, which adds ~20–24px per entry × 100+ entries. The section headers add ~200–400px of padding per section. Both changes are intentional improvements; this reports the density consequence.
- **Impact on user:** Existing mobile users navigate more scroll. New users have a clearer, less hidden lead-gen path. Net benefit.
- **Note for future:** If catalog entry count grows significantly, consider a virtualized or paginated view.

## Patterns & Systemic Issues

1. **Responsive images are the largest single unresolved technical liability.** Every page serves full-resolution images to every device. This was P1 in the previous audit and remains P1. The fix requires generating multi-resolution exports — a build step this project does not currently have. No CSS change resolves this; it requires asset pipeline work.

2. **Group B token replacement was complete for the 7 targeted values but did not fully extend to format-rail.css.** The token system is now strong for brand colors but format-rail.css still has structural UI values inline. Extend the token replacement pass to cover `--border-dark` and `--warm-ochre-light` usage in that file.

3. **The E-7 catalog WA link promotion created a new accessibility surface.** The CTA is now always visible but still has a 13px touch target. Touch target P1 was already present in the previous audit for this same element; promoting the element to always-visible makes fixing the touch target more urgent.

4. **Dead code from E-7 (consult button CSS + JS) should be removed in the next distill pass.** The toggle interaction was removed from the JS but its CSS rules (43 lines) and JS functions (30 lines) remain. This is cleanup, not a bug.

## Positive Findings

- **Brand coherence is dramatically improved.** All four main pages now feel designed by the same team. The two-visual-language problem (Editorial Atelier vs. Generic Web UI) identified in PHASE-0-VISUAL-AUDIT.md has been substantially resolved.
- **Token system is now complete and correct for brand colors.** The 8 new tokens (`--rouge-ink`, `--carmesi-blush`, `--ventisca-mist`, `--resina-dark`, `--warm-ochre-light`, `--warm-ochre-dark`, `--ink-on-dark`, `--ink-deep`) formalize the per-fragrance identity system that was previously hardcoded across 22+ locations.
- **Decants remains the strongest page and is visually unchanged.** The token replacement was a pure refactor — all per-fragrance colors and gradient overlays render identically. The benchmark page was preserved.
- **Coleccion desktop is premium and structurally intact.** The alternating editorial splits, dark section backgrounds, and warm-ochre category labels are working correctly. Format toggle warm-ochre selection is consistent.
- **Sessions section is now a genuine editorial threshold.** Dark volcanic-ink background with parchment text and a working `.btn--inv` CTA — the section now belongs to the same visual world as Decants.
- **Catalog section headers at display scale are a meaningful improvement.** The Diseñador / Nicho / Ultra Nicho headers create rhythm in what was previously a wall of identical text rows. The Ultra Nicho dark section with parchment display type is the strongest visual moment on the catalog page.
- **Scroll animations, a11y baseline, keyboard navigation, and focus indicators are all intact.** No regressions introduced in any of the four fix phases.
- **HTTP server enables accurate font rendering.** Screenshots are now captured with Google Fonts (Bitter + Manrope) loaded correctly, making the screenshots production-faithful.
