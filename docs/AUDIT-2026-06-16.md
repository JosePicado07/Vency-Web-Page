## Anti-Patterns Verdict

**Pass — not AI-generated.** The typography system (Bitter 900 + Manrope tracked labels), OKLCH palette, scroll entrances, and editorial image splits are distinctive and brand-coherent. No gradient text, no glassmorphism, no hero-metric template, no identical card grids, no drop shadows.

**One real concern:** The catalog page (catalogo.html) reads as a dense spreadsheet. 100+ rows of text with no imagery, no editorial breaks, and no visual weight differences between sections aside from background color. This is intentional (lead-gen list), but it conflicts with "slow is a stance" — first-time visitors who land directly on catalogo will not feel they are in an atelier.

## Detailed Findings

### P0 — Blocking

### P1 — Major (fix before release)

---

**[P1-A] Touch targets below minimum on mobile — catalog filter pills**
- **Location:** `src/styles/catalogo.css` lines 81–96 (`.cat-pill`)
- **Category:** Accessibility / Responsive
- **Impact:** Filter pills have `padding: 0.3rem 0.7rem` → ~12px rendered height at 16px base. WCAG 2.5.5 requires 44×44px minimum. On a 390px phone, these are untappable without precision. Users trying to filter "Nicho" or "Mujer" will misfire repeatedly.
- **WCAG:** 2.5.5 Target Size (AA)
- **Fix:** Increase pill padding to `0.65rem 1rem` minimum. Add `min-height: 2.75rem` (44px) and `display: inline-flex; align-items: center`.
- **Command:** `/impeccable adapt catalogo`

---

**[P1-B] Touch target below minimum — catalog WA / interp CTAs**
- **Location:** `src/styles/catalogo.css` lines 354–365 (`.cat-entry__wa`)
- **Category:** Accessibility / Responsive
- **Impact:** `.cat-entry__wa` has `padding: 0 0 0.8rem` — no top/side padding. The tappable area is the text width × line-height only (~10–12px tall). This is the primary lead-generation action on the catalog. On mobile, it is nearly impossible to tap accurately.
- **WCAG:** 2.5.5 Target Size (AA)
- **Fix:** Change padding to `0.6rem 0.8rem`. Change `display: inline-block` → `display: inline-flex; align-items: center`. Also add `min-height: 2.75rem`.
- **Command:** `/impeccable adapt catalogo`

---

**[P1-C] No responsive images — all pages**
- **Location:** All `<img>` tags across index.html, coleccion.html, decants.html (JS-rendered in coleccion.js)
- **Category:** Performance
- **Impact:** A 900×900px hero image (`hero-botanica.png`) is served to a 390px mobile screen. All 6 fragrance images (900px wide) are served at full resolution to every device. On a 4G mobile connection, this adds multiple seconds to meaningful render. No WebP or AVIF fallbacks at all.
- **Standard:** Core Web Vitals (LCP), Google PageSpeed
- **Fix:** Add `srcset` with 400w / 800w / 1200w variants per image. Convert to WebP. Use `<picture>` with AVIF source. For JS-rendered entries in coleccion.js, emit `srcset` in the `renderEntry()` img tag.
- **Command:** `/impeccable optimize`

---

**[P1-D] Catalog filter bar breaks on mobile (border-left orphan)**
- **Location:** `src/styles/catalogo.css` lines 69–73 (`.cat-filters__group + .cat-filters__group`)
- **Category:** Responsive
- **Impact:** The Género filter group uses `border-left: 1px solid var(--border)` + `margin-left: var(--space-md)`. When `flex-wrap: wrap` kicks in and the Género group drops to a new line, the `border-left` renders as a vertical rule floating at the far left of the new row, disconnected from anything. Looks broken.
- **Fix:** At the filter bar's wrap breakpoint (≤520px), change the second group's separator from `border-left` to `border-top` and reset the left margin/padding. Or stack both groups vertically with `flex-direction: column` at that width.
- **Command:** `/impeccable adapt catalogo`

---

**[P1-E] Gender dots encode information by color only**
- **Location:** `src/scripts/catalogo.js` (renderEntry), `src/styles/catalogo.css` lines 276–278
- **Category:** Accessibility
- **Impact:** The gender indicator for each catalog entry is a 5×5px colored dot (rose = mujer, blue = hombre, teal = unisex). No shape difference, no text label within the entry itself. The legend is only shown in the page header — for users who navigate directly to `#cat-listing` or use filter to reach a section, the legend is scrolled off-screen. Color-blind users (8% of males) cannot distinguish hombre from unisex.
- **WCAG:** 1.4.1 Use of Color (A)
- **Fix:** Add a visually hidden text label within each dot: `<span class="cat-entry__gender-dot" aria-hidden="true"></span><span class="sr-only">Hombre</span>`. Alternatively, use different dot shapes (circle/square/diamond). The legend in the header should also use shapes, not just colors.
- **Command:** `/impeccable harden catalogo`

### P2 — Minor

---

**[P2-A] Hardcoded OKLCH values bypassing the token system**
- **Location:** `src/styles/catalogo.css` (22 instances), `src/styles/decants.css` (12 instances)
- **Category:** Theming
- **Impact:** Values like `oklch(70% 0.09 55)`, `oklch(48% 0.085 55)`, `oklch(55% 0.015 180)` are used repeatedly — these are effectively `--warm-ochre`, `--warm-ochre-dark`, `--atelier-stone`, but spelled out inline. If a brand token is adjusted, these won't update. The decant gradient overlays also use raw OKLCH instead of token references.
- **Fix:** Define `--warm-ochre-dark: oklch(48% 0.085 55)`, `--warm-ochre-light: oklch(70% 0.09 55)`, `--ink-deep: oklch(55% 0.015 180)` in the `:root` block in styles.css. Replace all inline instances.
- **Command:** `/impeccable harden`

---

**[P2-B] Catalog page — no visual rhythm across 24,000px of entries**
- **Location:** `src/pages/catalogo.html`, `src/styles/catalogo.css`
- **Category:** Anti-Pattern (brand register)
- **Impact:** The catalog is a wall of identically-formatted text rows with no imagery, no editorial section breaks beyond background color, and no element to anchor the eye. On mobile it reaches 24,270px. The sections (Diseñador → Nicho → Ultra Nicho) change background but the entry format never changes. This is the opposite of "every fragrance is a story first" — here fragrances are line items.
- **Fix options (lightest to heaviest):**
  1. Add a full-width editorial label/count between sections: brand + count, large Bitter type, 80px of vertical breathing room.
  2. Make brand group headers visible (currently the section headers are just `<h2>` in small type). Give each letter grouping a larger label.
  3. Add one hero "featured" entry per section — larger text treatment, no image needed.
- **Command:** `/impeccable layout catalogo`

---

**[P2-C] JS-rendered images missing width/height attributes (CLS risk)**
- **Location:** `src/scripts/coleccion.js` — `renderEntry()` img tag
- **Category:** Performance
- **Impact:** When coleccion.js renders fragrance entries, the `<img>` tags it emits have no `width`/`height` attributes. The browser cannot reserve space before the image loads, causing Cumulative Layout Shift as each image loads in. This pushes content down and degrades Core Web Vitals score.
- **Fix:** Add `width="700" height="525"` (or the actual dimensions) to all JS-emitted img tags in `renderEntry()`. Also ensure `aspect-ratio` is set in CSS as fallback.
- **Command:** `/impeccable optimize coleccion`

---

**[P2-D] Catalog "Ver interpretación Vency" link has no visual aria-label for its context**
- **Location:** `src/scripts/catalogo.js` line 88–90
- **Category:** Accessibility
- **Impact:** The interp link renders as `<a class="cat-entry__interp" href="coleccion.html#ventisca">Ver interpretación Vency →</a>`. Without an `aria-label`, screen reader users hear "Ver interpretación Vency" with no indication of which fragrance. There are two such links on the page (Ventisca, Tabaco y Miel) — a screen reader cannot distinguish them.
- **WCAG:** 2.4.6 Headings and Labels (AA), 4.1.2 Name, Role, Value (A)
- **Fix:** Add `aria-label="Ver interpretación Vency de Silver Mountain Water"` (populated from `item.name`).
- **Command:** `/impeccable harden catalogo`

---

**[P2-E] Coleccion.html — filter pills not visible in initial viewport (sticky bar too small)**
- **Location:** `src/pages/coleccion.html`, `src/styles/catalog.css` (filter row)
- **Category:** Responsive / UX
- **Impact:** On both desktop and mobile screenshots, the filter pills row (All / Original Blend / Icon Series) appears below the page header as a thin line. At the screenshot scale they appear nearly invisible — the `font-size: 0.58rem` and minimal padding make this feel like a decorative separator, not an interactive filter. The ponytail audit already noted this (QW-5 discoverability issue).
- **Fix:** Add a persistent visible "FILTRAR" label or increase pill size to match the catalog filter pill size. Consider making the filter bar more visually distinct with a subtle background.
- **Command:** `/impeccable adapt coleccion`

**[P2-F] Sessions section on index — dead visual weight**
- **Location:** `src/pages/index.html` `.sessions` section
- **Category:** Brand register / UX
- **Impact:** The Sessions section is centered text on parchment, no imagery, no supporting visual. ## Patterns & Systemic Issues

1. **No responsive image pipeline.** The entire site serves fixed-resolution images with no `srcset`. This is the single biggest performance liability. Every page has it. Fixing it requires either a build step (sharp/imagemin) or manually exporting 400w/800w variants for each asset.

2. **Token system leaks.** The CSS custom property system (`--warm-ochre`, `--volcanic-ink`, etc.) is well-designed but not enforced in component-level CSS files. catalogo.css and decants.css both spell out OKLCH values directly that should reference tokens. As the token count grows, these inline values will drift out of sync.

3. **Touch targets consistently undersized in filter UI.** Both the catalog pills and the coleccion filter pills use the same undersized padding approach. This is a shared pattern that needs a shared fix — define a minimum `.pill` pattern with correct touch sizing once, apply everywhere.

## Positive Findings

- **Decants page is the strongest on the site.** Full-bleed editorial blocks, rich photography, per-fragrance color identity, clear format rail. This is what the brand register looks like working correctly. Replicate its visual confidence on other pages.
- **Coleccion desktop is premium.** The alternating editorial splits with large Bitter headlines are exactly right. The proportion of text to image is well-judged.
- **A11y baseline is solid.** Skip links, `aria-label` on all nav elements, `aria-expanded` on toggles, `aria-pressed` on filter pills, focus-visible outlines with ochre color — these are all present and correct.
- **Color system is distinctive.** Volcanic Ink / Botanical Parchment contrast is immediately legible as "not a generic brand." The Ultra Nicho dark section on catalogo works well as section differentiation.
- **Scroll animations are well-paced.** The `animate-in` stagger pattern with expo easing at 70–460ms delays feels unhurried and intentional. `prefers-reduced-motion` fallback is implemented.
- **Typography hierarchy is clean and consistent.** Bitter 900 for display, tracked Manrope uppercase for labels, Manrope 400 for body. No flat scale anywhere.
